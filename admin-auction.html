寄賣價格調整確認

用來整理寄賣商品價格調整、降價確認、再次上架前確認，勾選商品後產生給寄賣人確認的 PDF。

選寄賣人 ➜ 數量 > 0 改價格 / 流標通知 PDF / Excel 標籤 已預留後端更新 API

① 查詢 / 本次處理條件

正式接後端後：選寄賣人會抓 Products 裡「寄賣人相同、數量大於 0」的商品。
勾選商品時，系統會依目前流標次數自動帶入第一次 / 第二次 / 第三次
更改價格模式
勾選商品後，輸入新起標價格 / 新一標多少。按「更新資料庫價格」會送到後端 API 更新 Products 的「起標價格、一標多少」。PDF 會產出此寄賣人目前數量大於 0 的全部商品清單。
流標通知模式
勾選本次流標商品，選第幾次流標後產生 PDF。這個模式不會直接改價格，PDF 會提醒寄賣人：流標三次將寄回商品，不再寄賣。
這份 HTML 已經做好畫面、PDF、Excel 標籤、以及「更新資料庫價格」的前端送出位置。 但如果你的後端還沒有新增 API，按更新資料庫時會顯示後端尚未接好，不會亂改資料。

「更新資料庫價格」只應該用在更改價格模式。它會把勾選商品的新起標價格 / 新一標多少送到後端，正式接好後才會真的改 Products 的「起標價格、一標多少」。

③ PDF / Excel 標籤

PDF 走瀏覽器列印，可另存 PDF；Excel 標籤會匯出 Excel 可開啟的 .xls 檔。
Excel 欄位固定為：SKU、Name、寄賣人、起標價格、一標多少。
